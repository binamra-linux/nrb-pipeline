NRB-001 — Section 6.4 - Patch Management ICT Security Guidelines
Violation: trivy_high_cves

Immediate Fix

Update the base image to a currently supported version and pin to a specific tag.

FROM alpine:3.18 # or FROM ubuntu:22.04

NRB Justification

Using a current and supported base image ensures all its components are patched and do not include end-of-life software, thus complying with Section 6.4.

Risk if Ignored

Deployment of vulnerable base images could lead to exploitation of known critical CVEs, potentially compromising customer data and financial transactions handled by the fintech platform, resulting in regulatory penalties and loss of trust.
NRB-002 — Section 6.5 - Vulnerability Assessment ICT Security Guidelines
Violation: trivy_medium_cves

Immediate Fix

Update the base image and ensure all packages are up-to-date before building the image.

FROM your-base-image:latest RUN apt-get update && apt-get upgrade -y

NRB Justification

This action directly addresses the requirement for regular vulnerability assessments and remediation by upgrading packages to include available fixes for detected CVEs.

Risk if Ignored

An unpatched vulnerability could be exploited to compromise customer financial data, disrupt payment processing, or lead to fraudulent transactions, directly impacting the integrity and trust of the fintech service in Nepal.
NRB-002 — Section 6.5 - Vulnerability Assessment ICT Security Guidelines
Violation: trivy_unfixed_cves

Immediate Fix

Add a command to update and upgrade all packages in the Dockerfile before installing application dependencies.

RUN apt-get update && apt-get upgrade -y && rm -rf /var/lib/apt/lists/*

NRB Justification

This change ensures that the base image is updated to include security patches for known vulnerabilities, directly addressing the requirement for regular vulnerability assessment and remediation.

Risk if Ignored

Unpatched vulnerabilities could be exploited by attackers to gain unauthorized access to sensitive customer financial data (e.g., bank account details, transaction history) or disrupt payment processing services, leading to financial losses and reputational damage within Nepal's payment ecosystem.
NRB-010 — Section 4.4 - Technology Risk Risk Management Guidelines
Violation: trivy_os_cves

Immediate Fix

Add a command to update and upgrade packages at the beginning of the Dockerfile and specify a package version when installing a potentially vulnerable library.

RUN apt-get update && apt-get upgrade -y --no-install-recommends && rm -rf /var/lib/apt/lists/* RUN apt-get install -y --no-install-recommends your-package-name=1.2.3

NRB Justification

Regularly updating packages and pinning specific, known-good versions directly addresses the identification and management of risks from outdated libraries and open-source components.

Risk if Ignored

An unpatched vulnerability in a payment processing library could allow attackers to intercept or manipulate financial transactions, leading to customer data breaches, financial fraud, and severe reputational damage for the fintech startup.
