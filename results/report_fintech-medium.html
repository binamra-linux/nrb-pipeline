NRB-006 — Section 4.2 - Secure Communication Digital Payment Security Guidelines
Violation: trivy_openssl_cves

Immediate Fix

Update the base image to a version that includes patched OpenSSL/libssl libraries or explicitly install/update OpenSSL packages.

FROM ubuntu:22.04 RUN apt-get update && apt-get install -y --only-upgrade openssl libssl-dev && rm -rf /var/lib/apt/lists/*

NRB Justification

Updating OpenSSL and related libraries ensures that all communication protocols relying on them, like TLS, are utilizing the latest security patches, thereby encrypting communication as required by Section 4.2.

Risk if Ignored

Sensitive customer financial data (e.g., account numbers, transaction details, personal identification information) transmitted between payment components could be intercepted and compromised by malicious actors, leading to financial fraud, identity theft, and significant reputational damage for the fintech startup.
